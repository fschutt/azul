## Findings

### [MEDIUM] Inconsistent `into_option` signatures across `impl_option!` variants
- **Location**: `macros.rs:958` vs `macros.rs:977` vs `macros.rs:996`
- **Details**: The `copy = false, clone = false` variant takes `self` by value (`pub fn into_option(self)`), while the other two variants take `&self` (`pub fn into_option(&self)`) and clone internally. The `into_` prefix conventionally implies ownership transfer, making the `&self` versions misleading.
- **Recommendation**: Rename the `&self` versions to `to_option()` or make all variants consistent.
