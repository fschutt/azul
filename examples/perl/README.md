# Azul — Perl

✅ **Full GUI works.** `hello-world.pl` is the complete counter demo
(label + "Increase counter" button); it passes the headless counter
E2E (`scripts/e2e_language_matrix.sh perl` → `✓ WORKS`, 5 → 6 → 8).

Callbacks route through the host-invoker layer in `lib/Azul.pm`
(generated by `doc/src/codegen/v2/lang_perl/`). Three things make it
work end to end:

1. **Invoker registration** — each per-kind `FFI::Platypus` closure is
   marked `->sticky` and registered by casting it to a raw C function
   pointer (`$ffi->cast('(sig)->void', 'opaque', $closure)`). Passing a
   closure object straight to an `'opaque'` parameter does NOT hand C a
   callable trampoline.
2. **Return writeback** — the invoker copies the callback's return value
   into the thunk's out-pointer using the return type's **real C size**
   (`Azul::_writeback`). The Perl records over-size tagged-union payloads,
   so copying `length($$record)` would overflow the out-pointer.
3. **LayoutCallback install** — `hello-world.pl` splices the registered
   `AzLayoutCallback` (cb + host-handle ctx) into
   `window_state.layout_callback` at its **true C offset**, found via a
   sentinel probe (`AzWindowCreateOptions_create(<sentinel>)` +
   byte-scan). This bypasses the over-sized Perl union records rather
   than trusting their field offsets.

## Requirements

- Perl 5.30+ (Homebrew `/opt/homebrew/bin/perl` recommended)
- `FFI::Platypus` 2.x

## Run

```sh
AZ_LIB_DIR=. /opt/homebrew/bin/perl -Ilib hello-world.pl
```

`AZ_LIB_DIR` points the loader at the directory holding
`libazul.dylib`. Headless E2E:

```sh
AZ_E2E=../../tests/e2e/hello_world_counter.json AZ_BACKEND=headless \
  AZ_LIB_DIR=. /opt/homebrew/bin/perl -Ilib hello-world.pl
```

(System Perl on macOS lacks write permission to its site_perl;
install Platypus into Homebrew Perl.)

## Files

- `hello-world.pl` — full-GUI counter demo (layout + on-click callback).
- `lib/Azul.pm` — generated bindings (`Azul::` idiomatic + `Azul::FFI::` raw).
- `libazul.dylib` — prebuilt native library.

## Known codegen limitation

The Perl type layer emits tagged unions (e.g. `AzOptionRefAny`) as a
fixed `tag + uint8[256]` blob, so any Perl record that embeds a union
is larger than its C struct and has wrong internal field offsets. The
binding works around this where it matters (the writeback copies the
real size; the layout-callback splice uses the sentinel-probed offset).
Passing arbitrary union-containing structs by value field-by-field from
Perl is still unreliable until the union records are sized correctly.
