Generated by `azul-doc autoreview summarize-highlevel` from the review reports on
`midlevel-fixes-reference` (8f6d162f41a1). Each bullet is one architectural item

## How to process this plan

The items are grouped by **subsystem / file ownership** so that **one agent per
group** can run in parallel without stepping on each other. Each `### Group`
below names the files it owns. Two groups never own the same primary file.

A few **hot files are shared** across groups (mostly localized to distinct
functions). When assigning agents, either (a) make each agent touch only the
specific functions / line-ranges named in its bullets, or (b) sequence the
groups that share a hot file. The shared files:

- `dll/src/desktop/shell2/common/event.rs` — Group 5 (clipboard `event.rs:1771-1783`),
  Group 6 (drag sync `event.rs:2545/2668`), Group 9 (input-interpreter `event.rs:3870`).
- `layout/src/window.rs` — Group 3 (text3 overflow guard `:6171`),
  Group 5 (selection `styled_runs` `:7206`), Group 9 (gpu_state `synchronize_scrollbar_opacity`).
- `core/src/events.rs` — Group 9 (`TextInput` EventData variant).
- `layout/src/solver3/getters.rs` & `fc.rs` — Group 2 (calc viewport-unit fix reads both).

The **"Deletion audit results"** section at the very end records the audit of the

## Deletion audit results (67 auto-applied mid-level commits)

A 10-agent audit re-checked **every deletion** across the 67 applied commits
against the current tree (`git grep` for callers, `api.json` for public-API
surface, `git log -S` for re-adds). **Verdict: no functionality regressions.**
Every deleted unit was either SUPERSEDED (deduplicated into a shared helper,
renamed, moved, or inlined — behavior preserved) or genuinely DEAD (zero
