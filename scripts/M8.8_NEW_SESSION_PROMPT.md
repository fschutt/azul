# M8.8 New Session — Replace M8.7c Hacks with Production-Quality Code

## What you're walking into

The azul web backend's wasm-side dispatch pipeline reached a major
milestone in M8.7c: hello-world's `on_click` callback runs as
remill-lifted WebAssembly in a browser-like environment and visibly
mutates a counter in shared linear memory.

But the demo got there by stacking ~19 hacks (catalogued in
`scripts/HACKS_REVIEW_2026_05_16.md`). Your job is to replace each
hack with production-quality code, in the order recommended at the
bottom of that doc.

**Don't break the working demo.** Verify after each fix with:

```bash
cargo build -p azul-dll --release --features "build-dll web web-transpiler" --no-default-features
cd examples/c && cc -o hello-world.bin hello-world.c -lazul -L../../target/release -I../../dll
DYLD_LIBRARY_PATH=../../target/release AZ_BACKEND=web://127.0.0.1:8800 ./hello-world.bin &
node /tmp/e2e.js     # see test below if missing
```

The `/tmp/e2e.js` test (recreate if missing):

```js
const http = require("http");
async function fetch_(p) { return new Promise((res, rej) => {
    http.get("http://127.0.0.1:8800" + p, r => {
        const c = [];
        r.on("data", x => c.push(x));
        r.on("end", () => res(Buffer.concat(c)));
        r.on("error", rej);
    });
}); }
(async () => {
    const html = (await fetch_("/")).toString();
    const initialCounter = parseInt(html.match(/<div id="az_1">(\d+)<\/div>/)[1]);
    const typeId = BigInt(html.match(/"type_id":"(\d+)"/)[1]);
    const miniUrl = html.match(/href="(\/az\/mini\.[^"]+)"/)[1];
    const cbUrl = html.match(/href="(\/az\/cb\/[^"]+)"/)[1];
    const miniBytes = await fetch_(miniUrl);
    const cbBytes = await fetch_(cbUrl);
    const table = new WebAssembly.Table({ initial: 64, element: "anyfunc" });
    const realEnv = { __indirect_function_table: table, __az_resolve_callback: () => 0xFFFFFFFF };
    const stubFor = n => /write_memory|barrier|exception_clear/.test(n) ? (() => {}) : (/(?:_64|_f64)\b/.test(n) ? (() => 0n) : (() => 0));
    const h = env => ({ get: (_, p) => typeof p === "string" ? (Object.prototype.hasOwnProperty.call(env, p) ? env[p] : stubFor(p)) : undefined, has: () => true });
    const { instance: miniI } = await WebAssembly.instantiate(miniBytes, { env: new Proxy({}, h(realEnv)) });
    const mini = miniI.exports;
    const memory = mini.memory;
    mini.AzStartup_init(0, 0);
    const modelPtr = mini.AzStartup_alloc(4);
    new DataView(memory.buffer).setUint32(modelPtr, initialCounter, true);
    const refanyPtr = mini.AzStartup_hydrate(Number(typeId & 0xFFFFFFFFn), Number((typeId >> 32n) & 0xFFFFFFFFn), modelPtr, 4);
    const cbEnv = { memory, __indirect_function_table: table };
    const { instance: cbI } = await WebAssembly.instantiate(cbBytes, { env: new Proxy({}, h(cbEnv)) });
    const cb = cbI.exports.callback;
    const dv = new DataView(memory.buffer);
    const infoPtr = mini.AzStartup_alloc(256);
    let last = initialCounter;
    for (let i = 1; i <= 7; i++) {
        const r = cb(BigInt(refanyPtr), 0n, infoPtr);
        const c = dv.getUint32(modelPtr, true);
        console.log("click " + i + ": Update=" + r + " counter " + last + " -> " + c + (c === last + 1 ? " OK" : " FAIL"));
        last = c;
    }
    process.exit(last === initialCounter + 7 ? 0 : 1);
})().catch(e => { console.error(e.stack); process.exit(1); });
```

Expected output after every fix: `Update=1 counter N -> N+1 OK` for 7 clicks.

## The recommended fix order (from `HACKS_REVIEW_2026_05_16.md`)

Each fix below cites the hack number from the review doc + the
specific file/symbol to change.

### Step 1 — Cross-platform symbol size + PLT resolution

**Hacks fixed:** #4 (fixed `LIFT_READ_WINDOW`), #5 (macOS-arm64-only
PLT chase), #6 (arm64-only tail-call rewrite).

**Approach:** add `goblin = "0.10"` (or the `object` crate) as a dep
of azul-dll under the `web-transpiler` feature. Parse the host
binary's symbol table at server startup:

- On macOS, find the Mach-O header via the linker-provided extern
  `_mh_execute_header`; walk LC_SYMTAB + LC_DYSYMTAB to get the
  symbol table + indirect symbol table; walk LC_SEGMENT_64
  sections to find `__TEXT.__stubs` + its `reserved1` indirect
  index.
- On Linux, use `dl_iterate_phdr` to walk loaded ELF objects;
  parse `.symtab` + `.dynsym` + `.plt`.
- Build two maps at startup:
  - `symbol_size: HashMap<usize, usize>` — addr → byte size
    (next-symbol-addr minus this one's addr).
  - `stub_to_real: HashMap<usize, usize>` — addr-of-stub → addr-of-
    real-symbol (so we don't need to disassemble stubs at all).

**Files to touch:**
- `dll/src/web/mod.rs` — replace `LIFT_READ_WINDOW` consumers with
  a `symbol_size_for(addr)` lookup; replace
  `resolve_macos_arm64_stub` + the dladdr fallback with a
  `stub_to_real` lookup that's identity for non-stub addrs.
- `dll/src/web/transpiler_remill.rs` — `rewrite_tailcall_wrapper`
  goes away (the byte rewrite is fixing a symptom; with proper
  symbol-size detection + PLT mapping, the lift of
  `AzRefCount_canBeSharedMut` would see the real fn at its real
  addr and not need the trampoline rewrite). If it turns out remill
  STILL bails on bare `b imm26`, add per-arch tail-call detection
  in a way that handles x86_64 `jmp imm32` too.
- `dll/Cargo.toml` — add the parser crate.

**Verify:** the demo still works on macOS arm64. Bonus: build on
Linux x86_64 (use Docker if you don't have a Linux box) and verify
the cb-instantiate node test runs there.

### Step 2 — Wire `AzStartup_dispatchEvent` for full event handling

**Hacks fixed:** #2 (only-click), #14 (bypassing dispatchEvent),
#16 (regex hit-test on `az_N`), #17 (fake addr↔table-idx map).

**Approach:** restore the full dispatch chain. The mini wasm
already has an `AzStartup_dispatchEvent` body (currently uses
`FAKE_REFANY_LO = 0x101`). Make it use the hydrated refany ptr.

Concrete changes:
- Add `AzStartup_setAppData(state: u32, refany_ptr: u32)` in
  `eventloop.rs` so JS can hand the hydrated ptr into mini's
  state. (Or extend `AzStartup_init`'s signature — but adding a
  new fn is less invasive.)
- In `AzStartup_dispatchEvent`, replace the `FAKE_REFANY_*`
  constants with the stored `state.app_data_ptr`. Pass it via
  `__az_call_indirect` as the two i64 halves of the ptr (lo +
  hi).
- Add a wasm-side `cb_fn_cache: HashMap<node_idx, fn_addr>`
  populated from the hydrated StyledDom (Step 3 below). The
  `__az_resolve_callback` JS bridge then becomes a real
  fn-addr → table-idx lookup instead of identity.
- In `loader_js.rs`:
  - Delete `azInvokeCbDirect`. Restore `azDispatch` (kept in git
    history at commit `7530483aa^`).
  - Restore `azWireListeners` to bind every event kind via
    `azDispatch`.
  - Call `mini.AzStartup_setAppData(state, refanyPtr)` after
    hydrate.
  - The table-idx → cb mapping comes from `azNodeCbFns` already
    populated at bootstrap; that's fine as long as
    `__az_resolve_callback` looks up correctly.

**Verify:** click increments the counter (existing test); plus
manually verify in a browser that hover/focus listeners fire
(observable via the console.debug lines in the dispatch path).

### Step 3 — Lifted user `_fromJson` hydration path

**Hacks fixed:** #11 (`AzStartup_hydrate` hand-rolls RefAny), #12
(JS `data_size = 4` hardcoded), #18 (hello-world-only schema).

**Approach:** instead of writing RefCountInner fields directly,
invoke the user's lifted `_fromJson` deserializer.

Concrete changes:
- On the server side, find the user's `_fromJson` fn-ptr (it's
  registered on the RefAny via the AZ_REFLECT_JSON macro — read
  it from `app_data.sharing_info.downcast().deserialize_fn`).
- Add `_fromJson` to the cb wasm's transitive-lift roots (call
  `discover_and_transpile_callbacks` with an extra entry for it,
  OR build a separate lifted module specifically for the
  deserializer chain).
- Add an `AzJson` wrapper-signature variant in
  `signature_for_callback_kind` so the lifted `_fromJson` has a
  proper entry point (`fromJson(json_ptr: i32) -> resultrefany_ptr`).
- Build the AzJson aggregate JS-side OR add an
  `AzStartup_buildJson` helper to mini that takes raw bytes
  (postcard-encoded user data) and returns an AzJson ptr.
- In `loader_js.rs::azHydrate`: stop assuming `data_size=4`.
  Instead, base64-decode the embedded data bytes, copy into wasm
  memory, call AzJson-builder, call lifted `_fromJson`, get
  RefAny ptr back.
- Update `html_render.rs` to embed the user data as base64'd raw
  bytes (or postcard-encoded) in the az-hydrate payload, alongside
  the type_id.

**Verify:** swap MyDataModel in `examples/c/hello-world.c` to add
a string field, rebuild, ensure both fields hydrate + are visible
post-click.

### Step 4 — WASM-side hit-test + DOM diff/patch

**Hacks fixed:** #1 (DOM patch hardcoded to `az_1`), #16 (regex
hit-test), #19 (best-effort stylesheet).

**Approach:** ship the StyledDom into wasm via the hydration
payload, then run hit-test + diff inside `AzStartup_dispatchEvent`.

Concrete changes:
- Add an `AzStyledDom_serialize` / `AzStyledDom_deserialize` pair
  to libazul (probably via postcard). The serializer writes:
  - DOM tree (node hierarchy)
  - Computed CSS per node
  - Laid-out bounding boxes (so hit-test against geometry, not
    nesting)
- Embed the serialized StyledDom bytes in the HTML head as
  `<script id="az-styled-dom" type="application/octet-stream"...>`
  (base64'd).
- Wasm-side: `AzStartup_init` deserializes the StyledDom + caches
  it as `state.current_dom`.
- `AzStartup_dispatchEvent`: walk current_dom's bounding boxes
  for the (x, y) hit-test → identify node_idx → look up cb in
  cb_fn_cache → invoke.
- After cb returns Update::RefreshDom: invoke the lifted layout
  cb → produce new StyledDom → diff against old → emit TLV
  patches (the existing `azApplyPatches` handles SetText; add
  more kinds as needed — SetAttribute, AppendChild, RemoveChild).
- Loader.js stops doing JS-side `getElementById('az_1').textContent
  = ...`. Instead it just reads the patch buffer + applies via
  the existing TLV decoder.

For the styling part (item #3 in the review): bundled CSS that
the desktop renderer uses (e.g. azul-css's default theme)
should also be embedded into the rendered HTML's `<style>` block
so the button looks the same on first paint.

**Verify:** click in browser → counter visibly increments WITHOUT
the hardcoded `az_1` shortcut. Add a second clickable thing (e.g.
a "reset" button) and verify the right cb fires for the right
click.

### Step 5 — Generalize wrapper signatures from api.json types

**Hacks fixed:** #9 (canonical Callback shape for all wrappers).

**Approach:** the api.json has full type info per fn arg + return.
Build a per-sig wrapper generator that handles all PCS variants
(GprI64, GprI64Pair, GprPtr32, Wreg, HiddenPtrReturn for X8, ...).

Concrete changes:
- In `dll/src/web/transpiler_remill.rs`, replace
  `signature_for_callback_kind` with a generic
  `synthesize_wrapper_for_api_fn(fn_name: &str) -> CallbackSignature`
  that consults the embedded api.json's type info for `fn_name`.
- Add `Pcs::HiddenPtrReturn` for aggregates >16B (the wrapper
  takes an extra `out_ptr: i32` arg, stores into X8).
- Handle the AArch64 PCS rules properly:
  - ≤8B: single Wreg/GprI64.
  - 9-16B: GprI64Pair.
  - >16B: HiddenPtrReturn (caller-allocated buffer).
- For non-aggregate args (u32, i64, f64): use Wreg / GprI64 /
  the FP register slot.

**Verify:** wrap an api.json fn with a known signature (e.g.
`AzString_copyFromBytes(bytes: *const u8, start: usize, len:
usize) -> AzString`) using the new generator and confirm a JS
test can call it correctly.

### Step 6 — x86_64 + Linux support

**Hacks fixed:** #5 + #6 again (the byte-level pieces that Step 1
might have left as a TODO).

**Approach:**
- x86_64 PLT detection: `jmp qword ptr [rip + GOT_OFF]` pattern.
- x86_64 tail-call detection: `jmp imm32` (E9 ...) or
  `jmp [rip+disp32]` (FF 25 ...).
- Linux ELF .symtab + .got.plt parsing (the `goblin` or `object`
  crate handles this; just use it).
- Linux dispatch testing: spin up the demo in a Linux container.

**Verify:** the demo works on x86_64 Linux. Add a CI matrix entry
if there isn't one.

## What NOT to break

The following are intentional and correct:

- The **api.json walk** in `classify.rs` (drives off the brotli-
  compressed embedded JSON, no hand-coded fn list).
- The **bump allocator** in `transpiler_remill.rs::emit_helper_ir`
  (`@__az_bump_ptr` global, RustAlloc kind).
- **`--import-memory` + `--import-table`** for cb/layout wasms
  (mini owns memory, cb/layout share it).
- **PLT-stub THUNK emission** (current `sub_<stub> → musttail
  sub_<real>` thunks via `ResolvedBranchExtern.real_addr`) — this
  remains the right pattern even after Step 1 replaces the
  dladdr-based stub chase with a symbol-table lookup. Just
  populate `real_addr` from the lookup map instead of from the
  byte parse.
- **Pre-validation of RefAny serializer at startup** in
  `headless.rs::HeadlessApp::validate`.
- Server-side fallback for browsers without WASM support — when
  you implement Step 4, leave a `?fallback=server` URL param
  that takes the M0-M2 server-side dispatch path.
