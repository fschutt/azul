## Pre-requisites

- remill v6.0.0 available locally (per the experiment).

## Risks / open design questions

- **Lifting strategy**: remill is great at decompiling machine code, but the resulting LLVM IR is unfriendly to LLVM's optimizer (register-machine-shaped). May need a remill-specific cleanup pass. The blueprint experiment showed the basic path works; volume + variety of azul's functions may surface edge cases.
- **Floating-point precision**: WASM doesn't have x87 80-bit floats. Some Rust code may use `f64::INFINITY` / NaN-handling that lifts oddly. Spot-check after first hello-world succeeds.
- **Allocator boundary**: remill-lifted code expects libc-style `malloc`/`free`. The JS shim needs to provide a working WASM allocator (wee_alloc or dlmalloc-WASM). May want to use the same allocator the existing `azulc` crate is compiled with on `wasm32-unknown-unknown`.
